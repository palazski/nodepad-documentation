Research gets messy when every sub-question lives in the same transcript. NodePad works better when you split the work into clean branches, keep those branches visible, and then pull the useful pieces together deliberately — usually through a <EdgeChip type="merge">merge</EdgeChip> at the end.

## A good NodePad research pattern

<NumberedSteps
  items={[
    { title: 'Start with a framing branch', body: 'Use one branch to define the scope of the question and name the sub-questions you need to answer.' },
    { title: 'Branch by topic', body: 'Create one branch per sub-question so each line of inquiry has its own context and output history.' },
    { title: 'Reference across branches selectively', body: 'When one branch uncovers a source that matters elsewhere, use an `@` reference instead of summarizing it by hand.' },
    { title: 'Merge into a synthesis draft', body: 'Once the branches have enough signal, merge the important messages into a new draft and ask for a comparison or summary.' },
  ]}
/>

## Why it works

<FeatureGrid
  cols={3}
  accent="merge"
  items={[
    { title: 'Less prompt pollution', body: 'Each branch stays focused on one topic instead of inheriting every tangent.' },
    { title: 'Visible provenance', body: 'You can see where a conclusion came from because the source branches stay on the canvas.' },
    { title: 'Better synthesis', body: 'The final summary can draw from real source branches instead of from your memory of what happened in them.' },
  ]}
/>

<Callout tone="tip">
  Add a text note naming the research question and the standard for evidence you want to use. It helps when the workspace grows past a few branches.
</Callout>
