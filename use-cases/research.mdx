title: "Research"
description: "Keep separate investigations visible on one canvas and synthesize them only when you are ready."
---

Research gets messy when every sub-question lives in the same transcript. NodePad works better when you split the work into clean branches, keep those branches visible, and then pull the useful pieces together deliberately.

## A good NodePad research pattern

<Steps>
  <Step title="Start with a framing branch">
    Use one branch to define the scope of the question and name the sub-questions you need to answer.
  </Step>
  <Step title="Branch by topic">
    Create one branch per sub-question so each line of inquiry has its own context and output history.
  </Step>
  <Step title="Reference across branches selectively">
    When one branch uncovers a source that matters elsewhere, use an `@` reference instead of summarizing it by hand.
  </Step>
  <Step title="Merge into a synthesis draft">
    Once the branches have enough signal, merge the important messages into a new draft and ask for a comparison or summary.
  </Step>
</Steps>

## Why it works

<CardGroup cols={3}>
  <Card title="Less prompt pollution" icon="shield">
    Each branch stays focused on one topic instead of inheriting every tangent.
  </Card>
  <Card title="Visible provenance" icon="waypoints">
    You can see where a conclusion came from because the source branches stay on the canvas.
  </Card>
  <Card title="Better synthesis" icon="sparkles">
    The final summary can draw from real source branches instead of from your memory of what happened in them.
  </Card>
</CardGroup>

<Tip>
  Add a text note naming the research question and the standard for evidence you want to use. It helps when the workspace grows past a few branches.
</Tip>
