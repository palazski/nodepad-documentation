Engineering work benefits from clean alternate paths. One branch can hold the main diagnosis. Another can challenge it. A third can turn the decision into a rollout plan.

## A practical incident pattern

<Steps>
  <Step title="Start one branch with the raw problem">
    Put the logs, stack trace, or failing behavior into the first branch and ask for a diagnosis.
  </Step>
  <Step title="Branch before you act">
    Fork from the diagnosis point and ask another model or another prompt framing to challenge the first answer.
  </Step>
  <Step title="Use references for exact evidence">
    If one branch finds the crucial clue, reference that exact message in the branch where you are forming the plan.
  </Step>
  <Step title="Merge sources into the rollout draft">
    Merge the branches that matter into one new message and ask for the remediation plan, rollback plan, or test checklist.
  </Step>
</Steps>
