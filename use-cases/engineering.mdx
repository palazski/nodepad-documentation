Engineering work benefits from clean alternate paths. One branch can hold the main diagnosis. Another can <EdgeChip type="fork">fork</EdgeChip> from it to challenge the conclusion. A third can <EdgeChip type="merge">merge</EdgeChip> the useful pieces into a rollout plan.

## A practical incident pattern

<NumberedSteps
  items={[
    { title: 'Start one branch with the raw problem', body: 'Put the logs, stack trace, or failing behavior into the first branch and ask for a diagnosis.' },
    { title: 'Branch before you act', body: 'Fork from the diagnosis point and ask another model or another prompt framing to challenge the first answer.' },
    { title: 'Use references for exact evidence', body: 'If one branch finds the crucial clue, reference that exact message in the branch where you are forming the plan.' },
    { title: 'Merge sources into the rollout draft', body: 'Merge the branches that matter into one new message and ask for the remediation plan, rollback plan, or test checklist.' },
  ]}
/>
