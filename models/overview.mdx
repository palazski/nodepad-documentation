NodePad works with the two model families most teams already standardize on: Anthropic's Claude and OpenAI's GPT. You can switch between them at any point in a conversation — not just at the start of a thread — and run different models simultaneously across your canvas to compare outputs or delegate different tasks to the model best suited for each one.

<Note>
  NodePad adds new models as providers release them. The list of models you can select from updates automatically for all users.
</Note>

## What makes NodePad different

Most AI tools lock you into a single model for an entire conversation. NodePad removes that constraint in two ways.

**Per-message model selection.** You can change the model on any individual message, mid-thread. If a question calls for deeper reasoning, switch to Claude Opus for that message and switch back afterward. The rest of the thread is unaffected.

**Parallel model execution on the canvas.** Each thread on your canvas is independent. You can open one thread with Claude and another with GPT, ask them the same question, and read the answers side by side. No copy-pasting, no tab-switching.

<Tip>
  Running the same prompt across two models at once is one of the fastest ways to pressure-test an answer or spot where models disagree.
</Tip>
