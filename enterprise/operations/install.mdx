Install instructions for NodePad Enterprise are part of the deployment package you receive after the team scopes the right posture for your environment.

## How install works

<Steps>
  <Step title="Scope the deployment with the team" icon="mail">
    Share what your environment looks like — connectivity, target platform, identity, and capacity. We will recommend the right posture (connected, proxied mirror, fully air-gapped, or single-host pilot).
  </Step>
  <Step title="Receive the deployment package" icon="package">
    You get the install bundle, the configuration template, and a step-by-step install manual matched to your posture.
  </Step>
  <Step title="Walk through install with support" icon="users">
    A direct support channel stays open during the rollout. We help you fill in the configuration template, run the install, and verify the result.
  </Step>
</Steps>

## Postures we support

<Columns cols={2}>
  <Card title="Connected" icon="globe">
    Standard install where the target environment can reach the public internet.
  </Card>
  <Card title="Proxied mirror" icon="route">
    Install through your internal container registry mirror.
  </Card>
  <Card title="Fully air-gapped" icon="shield">
    No internet access at the install or operation stage. Distribution is packaged for your environment.
  </Card>
  <Card title="Single-host pilot" icon="box">
    Single machine for pilots, demos, or evaluation environments.
  </Card>
</Columns>
